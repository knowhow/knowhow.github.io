2) knowhow je baziran na xTuple ERP-u

xTuple reference i OpenRPT guide:

http://do-we-know-how.bring.out.ba/xtuple-openrpt-users-guide

Arhitektura

xTuple arhitektura je 2-tier: klasični rich GUI klijent, database server

backend je PostgreSQL Database server

Intenzivno se koriste view-ovi, stored procedure, organizacija baze database schema-ma.

https://github.com/knowhow/xtupleserver

Rich gui client

Core klijenta je realizovano u Qt, C/C++

Source code se može pogledati ovdje https://github.com/knowhow/xtuple/branches

Sadržaj pojedinih branche-ova:

  • master - upstream projekt
  • knowhow, MultiWHs - promjene rađene unutar knowhow ERP

xTule paketi - javascript

xTuple je iskoristio qtscript feature Qt-a za realizaciju aplikativnih modula koji oni zovu "packages".

Prazan paket koji zorno pokazuje strukturu jednog xtuple/knowhow paketa:

https://github.com/knowhow/skeleton

Primjer kompletnog programskog modula (POS modul) implementiranog kao package:

https://github.com/knowhow/xtpos/tree/knowhow

Reporting system - OpenRPT

Gore je data referenca na OpenRPT User's manual.

Fetchiranje podataka se vši sa MetaSQL-om. Priča se svodi na najobičnij SQL. Meta SQL omogućava jednostavnu razmjenu parametara između formi i reporta.

Lijepo riješeno.

UI Forme

Klasične Qt forme koje se prave sa Qt Designer-om. 

Behaviour se implementira  u javascript-u (packages) ili kao C++ Qt na nivou xtuple core-a.

Zamisao je da aplikativni developer nema potrebu da radi na C/C++ nivou.

xtops, xtte package-i kazuju da je realno moguće postići

Package update system

Bitno je naglasiti se instalacija paketa vrši sa jednog mjesta (updater aplikacija). Sve komponente se instaliraju u bazu.

Rezultat je taj dobijamo efekat deployment-a sličan web klijentu: instalacija se vrši na server, svi klijenti "kupe" update paketa sa servera.

xTuple gui client u brojkama

4) M3 mobilni klijent, A4 - aplikacioni server

M3 je Titanium appcelerator mobilni klijent (naša ciljna platforma Android, ali appcelerator podržava i IOS)

A4 je server-side dio mobilnog klijenta, ali je odmah predviđen kao aplikativni server knowhow.

Implementiran kao node.js server

Evo onoga što je do sada realizovano: http://knowhow.github.com/A4_knowhow/

Radi se o funkcijama koje putem JSON-a preuzimaju parametre aplikacije i listu artikala (slike, cijene) sa servera. 

M3, A4/M3b u konačnici trebaju implementirati sistem mobilne prodaje. 

S obzirom da imamo aktuelan ugovor sa klijentom, primarni cilj bring.out je implementacija sistema mobilnih narudžbi za veletrgovca cigarama.

Trenutno stanje:

  • Prototip M3 napravljen
  • Osnovne funkcije A4/M3b su realizovane. 

Reporting dio nije realizovan. Plan je da se taj dio implementira kao xtuple/knowhow javascript package. 

Razvoj M3 započet je na mobile-dev projektu.

Saša je napravio jedan prolaz kroz screen-ove aplikacije.
